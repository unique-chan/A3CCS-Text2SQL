* 아군은 'b', 적군은 'op'.

* 질문을 SQL로 변환하기 전에, 질문이 목록 조회인지 집계(aggregation)인지 구분하라.
* JOIN ... ON ... 은 반드시 명확한 외래키 관계를 기반으로 수행하고, 절대 추측으로 join 키를 만들지 않는다.
* 중복 결과가 발생하면, 먼저 join 조건이 올바른지 확인하고, 최후의 수단으로 DISTINCT를 사용하라.
* NULL 비교는 반드시 IS NULL 혹은 IS NOT NULL을 사용하고, =NULL은 절대 사용하지 않는다.
* 0으로 나누는 상황이 예상되면 조건을 추가하라.
* 질문이 모호하면 가장 보수적이고 안전한 해석을 따른다.
* SQL 쿼리를 생성하기 전에, 이 쿼리가 질문에 정확히 대응하는지 확인하라.
* 전투 결과를 추측하거나 미래의 움직임을 예상하지 말고, 오로지 객관적으로 데이터 필드만 참고하여 분석하여야 한다.

* .json 파일들은 특정 시점의 상태(snapshot)이고, 여러 개의 타임스탬프가 존재하지 않는다면 시간 흐름을 추론하지 마라.
* 현재 위치인 pos와 목표 위치인 waypointpos를 구분하라.
* damage=0이 손상이 없는 정상 상태이다.
* hitpoint는 3개의 병렬 배열 구조이다.
* discoverd=0이 발견되지 않은 상태이다.
* source는 메타데이터 파일 정보이며, 이를 data entity로 해석해서는 안 된다.
